# Decisiones del 2026-01-16

### [18:45] Sistema de marcado estructural v3.2 - Definición completa y validación

Se completó el desarrollo del sistema de marcado estructural para transformar el contenido del curso que ya tiene tono LcC aplicado. Este sistema define cómo organizar y marcar el contenido para facilitar su procesamiento, navegación y eventual transformación a otros formatos.

#### Contexto

Tras la aplicación masiva del tono LcC (v9) a los 37 archivos del curso, los ficheros mantenían:
- Separación artificial #teoria / #explicacion
- Sección "Notas para revisión" pendiente de eliminar
- Ausencia de marcado para recursos didácticos (metáforas, ejemplos, referencias)

#### Decisiones del sistema

**1. Principio base: Sin tag = teoría**

Todo contenido sin marcado explícito es contenido teórico por defecto. Solo se marcan las excepciones: recursos didácticos, multimedia y citas.

**2. Fusión #teoria + #explicacion**

Los bloques separados se integran en flujo continuo siguiendo la secuencia narrativa:
Concepto → Mecanismo → Manifestación → Implicación práctica

**3. Regla de límites: Bloque = Párrafo**

Cada tag aplica al contenido inmediatamente siguiente hasta la próxima línea en blanco. Esto resuelve la ambigüedad de dónde empieza y termina cada recurso marcado.

**4. Catálogo de tags**

| Categoría | Tag | Definición |
|-----------|-----|------------|
| Multimedia | #imagen | Fotografías, ilustraciones, diagramas, GIFs |
| Multimedia | #video | Vídeos |
| Didáctico | #metafora | Analogía desarrollada que re-explica concepto mediante imagen explorable |
| Didáctico | #ejemplo | Caso concreto desarrollado que ilustra el concepto |
| Didáctico | #practica | Actividad experiencial con instrucciones para el lector |
| Didáctico | #pregunta | Pregunta reflexiva dirigida al lector |
| Didáctico | #cita | Cita textual extensa con atribución |
| Referencias | [^n] | Footnotes markdown para fuentes académicas |

**5. Criterios de marcado #metafora (refinados en v3.2)**

Marcar cuando:
- Ocupa bloque diferenciado
- Tiene función didáctica (re-explica el concepto)
- Ofrece escenario que el lector puede "explorar"
- **El desarrollo es de la imagen/escenario, no solo del concepto teórico**

NO marcar:
- Metáforas lexicalizadas ("mecanismo", "cascada", "piloto automático")
- Comparaciones puntuales sin desarrollo
- Expresiones figuradas dentro de frases
- **Imágenes que solo etiquetan el concepto** (ej: "autopista de hábitos" seguido de explicación teórica sin desarrollar elementos de la autopista)

**6. Criterios de marcado #ejemplo**

Marcar cuando:
- Ocupa bloque diferenciado
- Presenta caso con desarrollo (contexto, acción, resultado)
- Ilustra claramente el concepto previo

NO marcar:
- Enumeraciones ilustrativas ("reuniones, tráfico, correos...")
- Menciones nominales sin desarrollo
- Listas de instancias sin narrativa

**7. Referencias académicas**

- Usar footnotes [^n] para cualquier fuente mencionada en el texto
- **Regla anti-invención:** Solo crear footnotes para fuentes explícitamente mencionadas. NUNCA inventar datos bibliográficos.

**8. Regla de abstención**

Si hay duda sobre si un recurso cumple los criterios, NO marcar. Es preferible omitir un recurso legítimo que marcar incorrectamente.

#### Jerarquía markdown

| Nivel | Uso |
|-------|-----|
| ## | Secciones principales (no modificar) |
| ### | Subsecciones existentes (no modificar) |
| #### | Subdivisiones internas (añadir donde proceda) |
| **negrita** | Ideas clave (2-3 por sección máximo) |

### [21:45] Prompt fusión-marcado v3.5 y checklist v1.2 - Correcciones de validación

Complementa: Sistema de marcado estructural v3.2

Durante la validación del Módulo 01 se detectaron deficiencias en la aplicación del prompt que requirieron correcciones.

#### Problemas detectados

1. **Footnotes en descripciones multimedia**: Las descripciones de #imagen que mencionaban autores+libros+años no generaban footnotes (ej: Nobel de Economía en m01-07)

2. **Citas sin footnotes**: Bloques #cita con autor(año) no tenían footnote inline asociado (ej: definiciones de mindfulness en m01-09)

3. **Numeración desordenada**: Footnotes numerados según orden de adición, no de aparición en documento

4. **Revistas omitidas**: TIME Magazine, National Geographic, HBR no reconocidos como fuentes

#### Correcciones al prompt (v3.3 → v3.5)

| Versión | Cambio |
|---------|--------|
| v3.3 | Alcance footnotes incluye descripciones multimedia |
| v3.4 | Artículo de revista en tabla de fuentes; relación #cita con footnotes |
| v3.5 | Numeración de footnotes según orden de aparición |

#### Correcciones al checklist (v1.1 → v1.2)

- E5 Referencias ampliado con verificaciones específicas
- Nueva sección "Errores Comunes Detectados"
- Nueva sección "Instrucciones para Agentes" con requisitos CRÍTICOS
